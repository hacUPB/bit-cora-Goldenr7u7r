# Actividad 3

**¿Qué esperaba ver en memoria (hipótesis)?**

Antes de ejecutar el experimento, esperaba ver que el objeto `ofApp` apareciera en memoria como una estructura con sus variables internas.

Pensaba que el depurador me iba a mostrar los datos que tiene el objeto, como el vector `particles`, y que todo eso estaría guardado dentro de una sola instancia.

**¿Qué pude observar?**

Al ejecutar el programa y usar el depurador, sí se puede ver el objeto `ofApp` en memoria mientras la aplicación está corriendo.

El depurador muestra sus atributos, y en este caso se puede ver el vector `particles` con los objetos que contiene. También se puede expandir y ver más detalles de lo que hay dentro.

**¿Qué información me proporciona el depurador?**

El depurador permite ver cómo está organizado el objeto en memoria, mostrando sus variables internas y los valores que tienen en ese momento.

También deja ver que el objeto sí existe como algo real en memoria y no solo como código.

**¿Qué puedo concluir?**

Puedo concluir que los objetos en programación orientada a objetos sí ocupan un espacio en memoria y contienen sus datos internamente.

Además, el depurador es útil para ver cómo están construidos los objetos y qué información tienen mientras el programa se está ejecutando.
